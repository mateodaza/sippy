## What is Sippy?

Sippy is a WhatsApp-based payment platform that lets anyone send and receive digital dollars using only their phone number. **Today, it's a general-purpose payment tool** - think Venmo, but via WhatsApp and powered by blockchain.

### How It Works

```
User sends WhatsApp message: "send 5 to +55 11 99999-9999"
     ↓
Sippy's AI understands the intent
     ↓
Blockchain transaction executes automatically
     ↓
Recipient gets notification instantly
```

**No app download. No seed phrases. No crypto jargon. Just WhatsApp.**

---

## What is Sippy Kids?

**Sippy Kids is a proposed educational fork of Sippy**, purpose-built for schools. It takes the proven technology (WhatsApp bot, AI, server wallets) and adapts it for safe, supervised financial education.

| | Sippy (Current) | Sippy Kids (Proposed Fork) |
|--|-----------------|---------------------------|
| **Purpose** | General payments | Financial education |
| **Users** | Anyone | Students, teachers, parents |
| **Currency** | PYUSD (real stablecoin) | Escola Coins (educational, zero value) |
| **Network** | Arbitrum Mainnet | Testnet (no real money) |
| **Supervision** | None | Parent/teacher oversight required |
| **Withdrawals** | To any wallet | Closed-loop only |
| **Open Source** | Proprietary | MIT License |

> **Infrastructure Isolation:** Sippy Kids runs on its own testnet infrastructure, completely separate from Sippy's mainnet funds and user base. No co-mingling of accounts, tokens, or data. Future mainnet integration (e.g., real scholarship disbursements) may be explored after legal and regulatory review.

**This partnership is about building Sippy Kids together** - not just deploying the existing Sippy in schools.

## Introducing: Sippy Kids Features

The Sippy Kids fork will include these educational features (built on top of proven Sippy infrastructure):

### 1. Educational Tokens (Not Real Money)

Deploy **"Escola Coins"** - educational tokens with zero financial risk:

- Teachers award coins for good grades, participation, behavior
- Students spend coins at school store, cafeteria, or with peers
- Real blockchain transactions, simulated economy
- All the learning, none of the risk

**Why Zero Risk?**
| Protection | How It Works |
|------------|--------------|
| **Testnet deployment** | Tokens exist on test blockchain with no monetary value |
| **Closed-loop system** | Coins only work within school ecosystem |
| **Non-redeemable** | Cannot be exchanged for real money or crypto |
| **No external transfers** | Students cannot send coins outside the system |
| **Controlled supply** | Only authorized teachers/admins can mint new coins |

**Example Flow:**
```
Teacher: "send 10 to +55 11 student-phone" (reward for homework)
Student: "send 3 to +55 11 classmate-phone" (buying a pencil)
Student: "balance" → "You have 7 Escola Coins"
```

*Note: This is a planned feature to be built in Phase 1 of the pilot.*

## Safeguarding & Compliance

Working with children requires the highest standards of data protection and safety. Here's our approach:

### Data Protection (LGPD/COPPA Aligned)

| Requirement | Our Approach |
|-------------|--------------|
| **Parental consent** | Required before any child account creation |
| **Age verification** | Parent/guardian must register child accounts |
| **Data minimization** | Only phone number stored; no names, photos, or PII on-chain |
| **Right to deletion** | Parents can request complete data removal within 30 days |
| **Retention window** | Phone numbers retained only while account is active; deleted 90 days after last activity or upon parent request |
| **No data selling** | Student data is never shared or monetized |

### On-Chain Privacy

- **No PII on blockchain** - Only wallet addresses and transaction amounts are recorded
- **Phone numbers** stored only in encrypted off-chain database
- **Pseudonymous** - Blockchain shows addresses, not identities
- **Testnet isolation** - Educational tokens on separate network from real funds

### AI Safety Guardrails

| Guardrail | Implementation |
|-----------|----------------|
| **Age-appropriate responses** | AI prompted to use simple, child-friendly language |
| **No financial advice** | AI teaches concepts, doesn't recommend investments |
| **Content filtering** | Responses screened for inappropriate content |
| **Topic boundaries** | AI stays within financial education scope |
| **Human escalation** | Complex questions flagged for teacher review |

### Incident Response

- **Designated contact**: safety@sippy.lat (Sippy safeguarding desk)
- **Partner escalation**: assigned BlockchainnaEscola lead during pilot kickoff
- **24-hour response** commitment for critical issues
- **Transparent reporting** to schools and parents
- **Regular security reviews** with BlockchainnaEscola

### Compliance Roadmap

| Phase | Action |
|-------|--------|
| Pre-pilot | Review with BlockchainnaEscola legal/compliance team |
| Pilot | Implement consent flows, test with small group |
| Scale | Third-party security audit, formal compliance certification |
